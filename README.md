# Discordコミュニティ向け 趣味マッチングアプリ（hobby matching）

## サービス概要

本サービスは、Discord上で既につながっている人同士の
「共通の話題」を可視化し、会話が始まる前の迷いや心理的ハードルを下げること
に特化した補助ツールです。

コミュニティ内のメンバー同士の共通趣味を一覧で把握でき、
誰と何の話題が合うのかを一目で確認できます。

---

## 想定ユーザー・利用シーン

本サービスは、主に Discord 上で
特定の話題に限定されず、多様な趣味を持つメンバーが集まる
小〜中規模の交流コミュニティを想定しています。

オンライン上で複数人と日常的に会話する一方で、
「誰とどの話題が合うのか分かりにくい」という課題を解決することを目的としています。

各ユーザーが、ゲーム・アニメ・漫画などの趣味を登録することで、
共通点を可視化し、会話のきっかけを見つけやすくします。

### 利用タイミング

本サービスは、毎日開くような SNS というよりも、
会話のきっかけを探したいときや、雑談・交流の前に確認するための補助ツール
としての利用を想定しています。

### 長期利用の価値

趣味を一度登録しておくことで、
新しく参加したメンバーや、久しぶりに話す相手との共通点を
すぐに把握できる点に価値があります。

---

## このサービスへの思い・作りたい理由

自分はゲームやアニメ、漫画が好きで、SNS上でもよく複数人と交流しています。  
その中で、

- 「この中で同じゲームを遊んでいる人は誰だろう?」
- 「共通のアニメの話題を振りたいけれど分からない」

と感じることが多くありました。

個別に聞くのは手間がかかり、話題を探している間に会話が流れてしまうこともあります。  
そこで、**趣味を事前に登録しておき、共通点を自然に見つけられる仕組み**があれば、  
SNS上での会話のハードルを下げられるのではないかと考え、このサービスを作りたいと思いました。

### X（旧Twitter）と本アプリを比較

Xの特徴は
- プロフィール文
- 投稿内容
- ハッシュタグ　
といった情報から、ユーザーの趣味や興味を間接的に読み取る必要があります。

課題として
- 「誰と何が共通か」が一覧では分からない
- 投稿を遡って把握する必要がある
- 会話のきっかけを自分で探す必要がある

本アプリでは、共通趣味を明示的に可視化させて
「会話の前に考えるコスト」を下げる補助ツールとなります。

### Discordと本アプリを比較

Discordの特徴は
- 趣味ごとのサーバー・チャンネル単位でのコミュニティ形成
- 会話はその場で完結するリアルタイム性の高い設計

課題として
- 「この人と何が共通か」は分かりにくい
- 人数が多いと関係性が見えづらい
- 初めて会話に参加する際の心理的ハードルが高い

本アプリでは、「このコミュニティ内で、誰と何が共通か」を先に可視化して、
Discord上の会話をスムーズにする“補助ツール”として機能します。

### 本アプリの特徴（まとめ）
SNSで既につながっている人同士の「共通の話題」を可視化し、
会話が始まる前段階での迷いや心理的ハードルを下げることに特化した補助ツールです

---

## サービスの利用イメージ

1. ユーザーがログインし、自分の趣味(ゲーム・アニメ・漫画など)を登録する
2. SNS上で繋がっている友達をアプリ内に登録する
3. 友達ごとに、共通している趣味が一覧で表示される
4. 「この人とはこのゲームが共通している」と分かり、SNS上での会話のきっかけとして活用する

※ 本サービス内でチャットは行わず、SNS上での交流を後押しする補助ツールとして利用される想定です。

### 補足:共通趣味の取得・判別フロー

1. ログインユーザーAが登録している趣味一覧を取得
   - User(ユーザー) と Hobby（趣味） は中間テーブル（UserHobby）を通じて関連付けられており、A が登録した趣味を一覧として取得します。

2. 友達として登録されたユーザーBが登録している趣味一覧を取得
   - 友達登録は一方向の参照関係（Friendship）として保持しており、A が登録した「友達ユーザーB」の情報から、趣味一覧を取得します。
   - 一方向にしているのは、友達登録は「人間関係」ではなく、「誰の趣味を見るか」という参照関係だからです。

3. 両者の趣味IDを比較し、共通するものを抽出
   - 取得した趣味同士を比較し、共通する趣味のみを抽出することで「ユーザーAとユーザーBの共通趣味」として判定します。

4. 判定結果をマイページ上に表示
   - 共通趣味が存在する場合は一覧表示し、存在しない場合は「共通の趣味はありません」と表示します。

---

## サービスの差別化ポイント・推しポイント

- SNSで既に繋がっている関係性を前提にしている点
- 共通の趣味を**自動的に可視化**できる点
- 会話や交流の「最初の一歩」を支援する点

単なる趣味管理ではなく、**人との会話を生み出すこと**にフォーカスしています。

- コミュニティ内での利用拡大
- 「共通の趣味を可視化できる」という分かりやすい価値の訴求

## 機能候補

### MVPリリース時に作りたい機能

① ユーザー登録・ログイン（最低限）
機能
- メールアドレス
- パスワード
- ニックネーム(必須、ユニーク、3〜20文字)

画面
- /users/sign_up
- /users/sign_in
-/mypage

実装ポイント
- Devise標準
- SNS連携なし
- プロフィール画像なし
- 自己紹介なし


② 趣味の登録（タグ方式）

登録方法
入力UI
- フリーテキスト1行
-	カンマ区切り

例：
Apex, モンハン, 呪術廻戦

保存ルール（中間試験_Railsのタグ機能のようなロジック）
- 空白除去
- 大文字小文字は区別しない（downcase）
- 既存Hobbyがあれば再利用
- なければ作成


③ 友達登録（ニックネーム指定・擬似SNS）

友達登録の仕様:
- 一方的な登録(Twitterのフォローのような形)
- 「この人の趣味を見る」ためだけの機能
- ブロック・承認なし

登録方法:

入力UI
- ニックネーム入力欄(1つ)
- 完全一致検索

画面
- /my_page/friends/new : 友達追加
- /my_page : 友達一覧(マイページに表示)

処理の流れ:
1. ニックネームを入力
2. 完全一致でユーザー検索

補足
のちに、承認制にする

1. ユーザーAがユーザーBを友達申請
2. ユーザーBが承認/拒否を選択
3. 承認されたら共通趣味が表示される



④ 共通趣味の一覧表示（MVPの核）
画面構成（例）
[マイページ]

▼ 友達一覧
--------------------------------

◉ user_A
  共通の趣味：
  - モンハン
  - Apex

◉ user_B
  共通の趣味：
  （なし）

--------------------------------

- 並び順：名前順
- 件数表示は不要
- 空の場合は「共通の趣味はありません」
